Duke Bot Setup

Follow the 3 steps below to get your bot running

1 Add Environment Variables in Vercel
Do this first. The bot token entered below only registers the webhook — it is not saved to the server. Add BOT_TOKEN and GOOGLE_API_KEY in Vercel so the bot can send replies and call the AI.
Protect your API keys. Without ALLOWED_USERS, anyone who finds your bot URL can send unlimited messages and drain your Gemini / OpenAI quota. Set ALLOWED_USERS to your Telegram ID — send /get_my_info to the bot to find it.
Open Vercel Dashboard → Settings → Environment Variables
Variable Value
BOT_TOKEN Your token from @BotFather REQUIRED
GOOGLE_API_KEY From Google AI Studio REQUIRED
ALLOWED_USERS Your Telegram ID or @username — send /get_my_info to get it. Comma-separate multiple users. ⚠ SECURITY
AUCH_ENABLE 1 = only ALLOWED_USERS can chat (default & recommended)
0 = open to everyone		 ⚠ SECURITY
OPENAI_API_KEY For OpenAI / OpenRouter / Groq / Mistral / etc. optional
OPENAI_BASE_URL Your provider's base URL (default: https://api.openai.com/v1) optional
ADMIN_ID Your Telegram user ID — enables admin commands optional

After adding env vars, click Redeploy in Vercel (Deployments → … → Redeploy).

2 Register the Webhook

Enter your bot token and click Set Webhook. This tells Telegram to forward messages to this deployment URL.

Set Webhook (start bot) Get Webhook Info Delete Webhook (stop bot)
3 Test & Secure Your Bot

Send /start to test. Then get your Telegram ID and add it to ALLOWED_USERS in Vercel to protect your API keys.

How to get your Telegram ID

Method 1 — Use this bot
Send /get_my_info to your bot → it replies with your user ID
Method 2 — @userinfobot
Open @userinfobot on Telegram and send any message — it replies instantly with your ID
For group chat IDs
Add your bot to the group and send /get_group_info — the bot replies with the group ID (a negative number like -1001234567890). Add it to ALLOWED_GROUPS.
