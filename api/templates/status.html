Duke Bot Setup

Follow the 3 steps below to get your bot running

1 Add Environment Variables in Vercel
Do this first. The bot token entered below is only used to register the webhook — it is not saved to the server. You must add BOT_TOKEN and GOOGLE_API_KEY in your Vercel project settings so the bot can send replies and call the AI.
Open Vercel Dashboard → Settings → Environment Variables
Variable Value
BOT_TOKEN Your token from @BotFather REQUIRED
GOOGLE_API_KEY From Google AI Studio REQUIRED
OPENAI_API_KEY OpenAI / OpenRouter / Groq / etc. optional
OPENAI_BASE_URL https://api.openai.com/v1 or your provider's URL optional
AUCH_ENABLE 0 = open to all, 1 = restricted (default) optional
ALLOWED_USERS Comma-separated Telegram IDs or usernames optional

After adding env vars, click Redeploy in the Vercel dashboard (Deployments → … → Redeploy).

2 Register the Webhook

Enter your bot token and click Set Webhook. This tells Telegram to forward messages to this deployment URL.

Set Webhook (start bot) Get Webhook Info Delete Webhook (stop bot)
3 Test Your Bot

Open your bot in Telegram and send /start. Use /get_my_info to get your user ID for ALLOWED_USERS if access control is enabled.
